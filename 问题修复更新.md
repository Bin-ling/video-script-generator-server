## 本次修复问题汇总

### 问题 1：分析模块 `llm` 空指针异常

**文件**: `app/services/analysis_service.py:241-303`

**问题**: `analyze_merged_modules` 函数直接调用 `llm.analyze_video_directly()`、`llm.analyze_image()` 等方法，但没有先检查 `llm` 变量是否为 `None`

**错误信息**:

```
'NoneType' object has no attribute 'analyze_video_directly'
'NoneType' object has no attribute 'analyze_image'
'NoneType' object has no attribute 'analyze_text'
```

**修复**:

- 在函数开头添加 `llm` 空值检查
- 为异常处理添加详细堆栈跟踪

***

### 问题 2：设置页面保存配置失败 - 前后端数据格式不匹配

**文件**: `app/services/settings_service.py:6-68`

**问题**:

- 前端发送: `{ models: { default: { api_key, base_url, model_name } } }`
- 后端期望: `{ api_key, base_url, model_name }`

**错误信息**: `Missing required fields`

**修复**:

- `save_model_config`: 支持两种数据格式，保存完整配置到 `model_config.json`
- `get_all_configs`: 返回前端期望的 `{ models: { ... } }` 格式

***

### 问题 3：历史页面视频加载失败

**文件**: `templates/history.html:1287`

**问题**: 视频路径处理不完整，只替换了一个反斜杠

**修复**: 使用正则表达式 `replace(/\\/g, '/')` 统一处理所有路径分隔符

***

### 问题 4：历史页面模块分析时重复加载媒体

**文件**: `templates/history.html:1215-1239, 1296-1325`

**问题**: 每个模块分析部分都会尝试加载视频或图片，造成不必要的资源消耗

**修复**:

- 移除每个模块的媒体加载代码
- 只在页面顶部显示一个主视频预览

***

### 问题 5：历史页面添加关键帧显示

**文件**: `templates/history.html:1292-1305, 1085-1093`

**问题**: 历史页面没有显示提取的关键帧

**修复**:

- 在主视频预览下方添加关键帧区域
- 添加 `loadFrames()` 函数调用
- 优化路径处理逻辑

***
